Slow but Straightforward

This section includes sorting algorithms sit in the domain of $\mathcal{O}(n^2)$. They are usually easy-understanding and have acceptable performance on small data sets. But when dealing with large data sets, the $\mathcal{O}(n^2)$ algorithms are usually not desirable.

The algorithms introduced in this sections are insertion sort and bubble sort.

Insertion Sort

Insertion sort is one of the most fundamental sorting algorithm. It works like sorting a hand of card: pick the next card and find the right location to insert it. One advantage of insertion sort is, when the list is "almost sorted", the instrtion sort can be very effective. This property can be applied by other sorting algorithms like Shellsort (Wikipedia page)

The python implementation of insertion sort is: 

                                def insertionSort(data):
                                    if data is None or data == []:
                                        return []
                                    for i in range(1, len(data)):
                                        currdata = data[i]
                                        j = i - 1
                                        while j >= 0 and data[j] > currdata:
                                            data[j + 1] = data[j]
                                            j -= 1
                                        data[j + 1] = currdata
                                    return data

Selection Sort

Selection sort is similar to insertion sort. At each iteration, it select the minimum element from the unsorted part of the list and put it in the end of sorted part of the list.

The python implementation of selection sort is: 

                                def selectionSort(data):
                                    if data is None or data == []:
                                        return []
                                    for i in range(len(data)):
                                        minidx, mindata = i, data[i]
                                        for j in range(i, len(data)):
                                            if data[j] < mindata:
                                                minidx = j
                                                mindata = data[j]
                                        if minidx >= 0:
                                            data[i], data[minidx] = data[minidx], data[i]
                                    return data

Bubble Sort

Bubble sort is also $\mathcal{O}(n^2)$ sorting algorithm. The algorithm keeps comparing the neighbored elements, if they are out of order then swap them. Although it has same time complexity as previous two algorithms, bubble sort is usually much slower than the previous two algorithms because it only moves elements one step at a time.

The python implementation of bubble sort is: 

                                def bubbleSort(data):
                                    if data is None or data ==[]:
                                        return []
                                    for i in range(len(data)):
                                        for j in range(1, len(data)):
                                            if data[j - 1] > data[j]:
                                            data[j - 1], data[j] = data[j], data[j - 1]
                                    return data

Fast but Tricky

This section includes sorting algorithms sit in the domain of $\mathcal{O}(n\log{}n)$. These algorithms usually require some brain muscle to be fully understood, but since they have very good performance, the time investment will be worth it especially when you are planning to sort gigantic data sets.

Sorting algorithms achieves $\mathcal{O}(n\log{}n)$ complexity usually adopted the divide and conquer technique. And this has been proved to be the best a comparing-based sorting algorithm can do.

The algorithms introduced in this sections are merge sort and quick sort.

Merge Sort

Merge sort divide the input list into sublists and merge them. If the sublist has length of 1, then the sort is complete, if the sublist has more than 1 elements, then merge sort the left sublist and merge sort the right sublist and merge them recursively. And the whole list is then sorted.

The python implementation of merge sort is: 

                                def mergeSort(data):
                                    if data is None or data == []:
                                        return []
                                    if len(data) == 1:
                                        return data
                                    mid = len(data) // 2
                                    a = mergeSort(data[:mid])
                                    b = mergeSort(data[mid:])
                                    return merge(a, b)

                                def merge(a, b):
                                    soln = []
                                    i = 0
                                    j = 0
                                    while i < len(a) and j < len(b):
                                        if a[i] <= b[j]:
                                            soln.append(a[i])
                                            i += 1
                                        else:
                                            soln.append(b[j])
                                            j += 1
                                    if i < len(a):
                                        return soln + a[i:]
                                    else:
                                        return soln + b[j:]

Quick Sort

                                def quickSort(data, start = 0, end = None):
                                    if data is None or data == []:
                                        return []
                                    if end is None:
                                        end = len(data) - 1
                                    if start < end:
                                        pivotIdx = partition(data, start, end)
                                        quickSort(data, start, pivotIdx - 1)
                                        quickSort(data, pivotIdx + 1, end)
                                    return data

                                def partition(data, start, end):
                                    pivot = data[start]
                                    i, j = start, start + 1
                                    while j <= end:
                                        if data[j] <= pivot:
                                            data[i + 1], data[j] = data[j], data[i + 1]
                                            i += 1
                                        j += 1
                                    data[start], data[i] = data[i], data[start]
                                    return i

Proceed with Caution!

In this section, I am trying to give you a taste of poorly (or sometime carefully) designed sorting algorithms with terrible performance. I will use stooge sort and the well-known bogo sort as an example.

Stooge Sort

Stooge sort is usually used as an example of poorly designed sorting algorithm. The algorithm itself is simple and but fairly inefficient. The time complexity of stooge sort is $$\mathcal{O}(n^{\frac{\log{3}}{\log{1.5}}})\approx\mathcal{O}(n^{2.71})$$ which is worse than bubble sort, selection sort and insertion sort.

The algorithm can be described relative easily:

  • Swap the first and last elements if the last one is smaller

  • If the length if list is greater or equal to 3, then:

    • Stooge sort the first two thirds of the list

    • Stooge sort the last two thirds of the list

    • Stooge sort the first two thirds of the list again

The python implementation of stooge sort is: 

                                def stoogeSort(data, start=0, end=None):
                                    if data is None or data == []:
                                        return []
                                    if end is None:
                                        end = len(data) - 1
                                    if data[end] < data[start]:
                                        data[end], data[start] = data[start], data[end]
                                    if (end - start + 1) > 2:
                                        step = (end - start + 1) // 3
                                        stoogeSort(data, start, end - step)
                                        stoogeSort(data, start + step, end)
                                        stoogeSort(data, start, end - step)
                                    return data

Bogo Sort

Bogo sort is a sorting algorithm which the complexity is solely depends on LUCK. The algorithm it self is easy, it tries to mimic the physical sorting a deck of card by repeatedly throwing the whole deck in the air, pick them up and check if they are sorted.

The random module of python has built-in function for shuffling a list. So the code for bogo sort can be easily implemented.

The best case complexity of bogo sort is $\mathcal{O}(n)$. This is trivial because it is possible to shuffle a list once and it's then sorted. The worst case complexity, unfortunately, is $\infty$, because you can always shuffle the list and get unsorted result. And according to Wikipedia the average time complexity of bogo sort is $\mathcal{O}((n+1)!)$.

The python implementation of bogo sort is: 

                                from random import shuffle
                                def isSorted(data):
                                    for i in range(len(data) - 1):
                                        if data[i] > data[i + 1]:
                                            return False
                                    return True

                                def bogoSort(data):
                                    while not isSorted(data):
                                        shuffle(data)
                                    return data

To show how quickly the time complexity of bogo sort grows, I tried to bogo sort lists with different lengths 1000 times and the average time consumption for each sort are: 

                                6 elements takes 2.62e-03 sec.
                                8 elements takes 1.94e-01 sec.
                                10 elements takes 2.06e+01 sec.

The program was running a linux box with quad-core i7 processor, bogo sort on lists with more than 10 elements for 100 times was not able to finish in 5 hours so I ended up hitting the ^c.
