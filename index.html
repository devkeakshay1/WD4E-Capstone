Proceed with Caution!

In this section, I am trying to give you a taste of poorly (or sometime carefully) designed sorting algorithms with terrible performance. I will use stooge sort and the well-known bogo sort as an example.

Stooge Sort

Stooge sort is usually used as an example of poorly designed sorting algorithm. The algorithm itself is simple and but fairly inefficient. The time complexity of stooge sort is $$\mathcal{O}(n^{\frac{\log{3}}{\log{1.5}}})\approx\mathcal{O}(n^{2.71})$$ which is worse than bubble sort, selection sort and insertion sort.

The algorithm can be described relative easily:

  • Swap the first and last elements if the last one is smaller

  • If the length if list is greater or equal to 3, then:

    • Stooge sort the first two thirds of the list

    • Stooge sort the last two thirds of the list

    • Stooge sort the first two thirds of the list again

The python implementation of stooge sort is: 

                                def stoogeSort(data, start=0, end=None):
                                    if data is None or data == []:
                                        return []
                                    if end is None:
                                        end = len(data) - 1
                                    if data[end] < data[start]:
                                        data[end], data[start] = data[start], data[end]
                                    if (end - start + 1) > 2:
                                        step = (end - start + 1) // 3
                                        stoogeSort(data, start, end - step)
                                        stoogeSort(data, start + step, end)
                                        stoogeSort(data, start, end - step)
                                    return data

Bogo Sort

Bogo sort is a sorting algorithm which the complexity is solely depends on LUCK. The algorithm it self is easy, it tries to mimic the physical sorting a deck of card by repeatedly throwing the whole deck in the air, pick them up and check if they are sorted.

The random module of python has built-in function for shuffling a list. So the code for bogo sort can be easily implemented.

The best case complexity of bogo sort is $\mathcal{O}(n)$. This is trivial because it is possible to shuffle a list once and it's then sorted. The worst case complexity, unfortunately, is $\infty$, because you can always shuffle the list and get unsorted result. And according to Wikipedia the average time complexity of bogo sort is $\mathcal{O}((n+1)!)$.

The python implementation of bogo sort is: 

                                from random import shuffle
                                def isSorted(data):
                                    for i in range(len(data) - 1):
                                        if data[i] > data[i + 1]:
                                            return False
                                    return True

                                def bogoSort(data):
                                    while not isSorted(data):
                                        shuffle(data)
                                    return data

To show how quickly the time complexity of bogo sort grows, I tried to bogo sort lists with different lengths 1000 times and the average time consumption for each sort are: 

                                6 elements takes 2.62e-03 sec.
                                8 elements takes 1.94e-01 sec.
                                10 elements takes 2.06e+01 sec.

The program was running a linux box with quad-core i7 processor, bogo sort on lists with more than 10 elements for 100 times was not able to finish in 5 hours so I ended up hitting the ^c.
